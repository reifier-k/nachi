# Security Policy / セキュリティポリシー

## Reporting a vulnerability

Do not report suspected vulnerabilities in a public issue. Use
[GitHub private vulnerability reporting](https://github.com/reifier-k/nachi/security/advisories/new)
and include the affected package and version or commit, impact, reproduction steps, and any known
mitigation. Do not include unrelated secrets or personal data.

The maintainer will validate the report privately, coordinate a fix and disclosure when applicable,
and credit the reporter unless anonymity is requested. Until nachi publishes stable releases,
security fixes target the latest `main`; no older compatibility line is supported.

## 脆弱性の報告

脆弱性が疑われる場合は公開Issueに投稿せず、
[GitHubの非公開脆弱性報告](https://github.com/reifier-k/nachi/security/advisories/new)を利用してください。
対象パッケージとバージョンまたはコミット、影響、再現手順、判明している緩和策を含め、無関係な秘密情報や
個人情報は記載しないでください。

メンテナーは非公開で内容を検証し、必要に応じて修正と公開時期を調整します。匿名希望がなければ報告者を
クレジットします。nachiが安定版を公開するまでは、最新の`main`のみをセキュリティ修正対象とし、古い互換
ラインはサポートしません。
